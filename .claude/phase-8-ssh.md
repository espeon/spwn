# phase 8: SSH gateway

**goal:** `spwn ssh <vm>` drops the user into a shell inside their VM. `spwn keys add/list/rm` manages their SSH public keys stored in spwn.

---

## overview

```
spwn ssh myvm
  → built-in SSH client (golang.org/x/crypto/ssh)
  → connects to ssh-gateway:2222 with username=<vm-name>, password=<bearer-token>
  → gateway calls control plane HTTP: POST /internal/gateway/auth
  → gateway calls control plane HTTP: GET /internal/gateway/vm?name=<vm>&account_id=<id>
  → gateway opens gRPC StreamConsole to the host agent with vm_id
  → host agent SSHes to VM at 172.16.x.2:22 using a platform keypair
  → bytes relay: user terminal ↔ gateway ↔ gRPC ↔ host agent ↔ VM SSH
```

---

## rootfs requirement

the base rootfs must have:
- `openssh-server` (or dropbear) installed and running via init
- `/root/.ssh/authorized_keys` containing the platform public key

the platform keypair is generated by the host agent at startup and stored at
`PLATFORM_KEY_PATH` (default: `/var/lib/spwn/platform_key`). after generating it,
the operator must add the public key to the rootfs image (rebuild or inject into
the overlay template). this is a one-time setup step documented in CLAUDE.md.

as a fallback for VMs where the user has installed and configured sshd themselves,
the relay still works — the host agent just needs a way to get in.

## components

### 1. DB migration — `ssh_keys` table

```sql
CREATE TABLE ssh_keys (
    id          TEXT PRIMARY KEY,
    account_id  TEXT NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
    name        TEXT NOT NULL,
    public_key  TEXT NOT NULL,
    fingerprint TEXT NOT NULL UNIQUE,
    created_at  BIGINT NOT NULL
);
```

queries: `add_ssh_key`, `list_ssh_keys`, `get_ssh_keys_for_account`,
`delete_ssh_key`, `get_account_by_key_fingerprint` (for pubkey SSH auth).

### 2. Control plane — internal gateway endpoints

protected by `Authorization: Bearer <GATEWAY_SECRET>` (shared secret,
never exposed to users).

```
POST /internal/gateway/auth/password
  body: { username: string, password: string }
  → verifies password OR bearer token against accounts + api_tokens
  → returns: { ok: bool, account_id: string, error?: string }

POST /internal/gateway/auth/pubkey
  body: { fingerprint: string }
  → looks up account by SSH key fingerprint
  → returns: { ok: bool, account_id: string, error?: string }

GET  /internal/gateway/vm?name=<name>&account_id=<id>
  → looks up VM by name for account, returns host agent gRPC address + VM IP
  → returns: { vm_id, host_agent_addr, vm_ip, status, exposed_port }
```

the control plane reads `GATEWAY_SECRET` at startup (required if gateway is enabled,
can be any random string). the gateway reads the same secret to authenticate its calls.

### 3. Control plane — SSH keys HTTP API

```
GET    /api/account/keys       → list SSH keys for authenticated user
POST   /api/account/keys       → add a key { name, public_key }
DELETE /api/account/keys/:id   → remove a key
```

response type:
```json
{ "id": "...", "name": "...", "fingerprint": "...", "created_at": 123456 }
```

no VmOps changes needed — keys belong to the account, not VMs.

### 4. Proto change — `ConsoleInput` gets `vm_id`

```protobuf
message ConsoleInput {
  string vm_id = 1;  // set only in the first message; empty for subsequent data frames
  bytes  data   = 2;
}
```

the gateway sends one ConsoleInput frame with `vm_id` set and empty `data`,
then streams data frames with `data` set and empty `vm_id`.
the host agent reads vm_id from the first frame, ignores it on subsequent frames.

### 5. Host agent — `StreamConsole` implementation

```rust
async fn stream_console(
    &self,
    req: Request<Streaming<ConsoleInput>>,
) -> Result<Response<Self::StreamConsoleStream>, Status> {
    // 1. read first frame to get vm_id
    let mut stream = req.into_inner();
    let first = stream.next().await...;
    let vm_id = first.vm_id;

    // 2. look up VM IP from DB
    let vm = db::get_vm(&self.pool, &vm_id)...;
    let vm_ip = vm.ip_address; // e.g. "172.16.1.2"

    // 3. load platform private key from PLATFORM_KEY_PATH
    let key = load_platform_key()?;

    // 4. open SSH connection to vm_ip:22 as root with platform key
    let tcp = tokio::net::TcpStream::connect(format!("{vm_ip}:22")).await?;
    let session = open_ssh_session(tcp, key).await?;
    let channel = session.request_pty_and_shell().await?;

    // 5. relay: incoming gRPC stream → SSH channel stdin
    //          SSH channel stdout → outgoing gRPC stream
    relay_bidirectional(stream, channel)
}
```

platform key loading:
```rust
fn load_platform_key() -> anyhow::Result<ssh_key::PrivateKey> {
    let path = std::env::var("PLATFORM_KEY_PATH")
        .unwrap_or_else(|_| "/var/lib/spwn/platform_key".into());
    if !Path::new(&path).exists() {
        // generate once
        let key = ssh_key::PrivateKey::random(&mut rand::rngs::OsRng, ssh_key::Algorithm::Ed25519)?;
        std::fs::write(&path, key.to_openssh(LineEnding::LF)?)?;
        info!("generated platform SSH key at {path}");
        info!("public key: {}", key.public_key().to_openssh()?);
    }
    Ok(ssh_key::PrivateKey::read_openssh_file(Path::new(&path))?)
}
```

deps to add to host-agent: `ssh_key`, `thrussh` or `russh` (async SSH client).

### 6. SSH gateway binary — `cmd/ssh-gateway`

```
services/cmd/ssh-gateway/
  main.go          config loading, start server
  server/
    server.go      wish SSH server setup, middleware
    auth.go        password + pubkey auth handlers → HTTP to control plane
    handler.go     session dispatch (shell relay)
    relay.go       bidirectional I/O between SSH channel and gRPC stream
  grpc/
    client.go      gRPC client to host agents (StreamConsole)
  config/
    config.go      env var parsing
```

**config env vars:**
```
SSH_GATEWAY_LISTEN_ADDR   :2222
SSH_GATEWAY_HOST_KEY_PATH /var/lib/spwn/ssh_gateway_host_key  (generated if absent)
CONTROL_PLANE_URL         http://localhost:5000   (gRPC for StreamConsole to host agents)
CONTROL_PLANE_HTTP_URL    http://localhost:3019   (HTTP for gateway auth calls)
GATEWAY_SECRET            <shared secret>
```

**auth flow (wish middleware):**
```go
// password auth
func handlePasswordAuth(ctx ssh.Context, password string) bool {
    resp := postToControlPlane("/internal/gateway/auth/password",
        map[string]string{"username": ctx.User(), "password": password})
    if resp.OK {
        ctx.SetValue("account_id", resp.AccountID)
        return true
    }
    return false
}

// pubkey auth
func handlePublicKeyAuth(ctx ssh.Context, key ssh.PublicKey) bool {
    fp := ssh.FingerprintSHA256(key)
    resp := postToControlPlane("/internal/gateway/auth/pubkey",
        map[string]string{"fingerprint": fp})
    if resp.OK {
        ctx.SetValue("account_id", resp.AccountID)
        return true
    }
    return false
}
```

**session handler:**
```go
func handleSession(s ssh.Session) {
    accountID := s.Context().Value("account_id").(string)
    vmName := s.User()  // convention: ssh myvm@spwn.run

    // look up VM
    vm := getVMFromControlPlane(vmName, accountID)
    if vm == nil {
        fmt.Fprintf(s, "vm %q not found\r\n", vmName)
        s.Exit(1)
        return
    }
    if vm.Status != "running" {
        fmt.Fprintf(s, "vm %q is not running (status: %s)\r\n", vmName, vm.Status)
        s.Exit(1)
        return
    }

    // request PTY
    ptyReq, winCh, isPty := s.Pty()

    // open StreamConsole gRPC stream to host agent
    stream := openConsoleStream(vm.HostAgentAddr, vm.VMID, ptyReq.Term, ptyReq.Window)

    // relay bytes
    go func() { io.Copy(grpcSink(stream), s) }()
    go func() { io.Copy(s, grpcSource(stream)) }()
    // handle window resize
    go handleWindowChanges(winCh, stream)

    <-stream.Done()
}
```

the gateway has no bubbletea TUI in phase 8 — it's purely a shell relay.
the TUI dashboard can come in a later phase once shell relay works end-to-end.

### 7. CLI — fill in stubs

**`spwn ssh <vm>`** — built-in SSH client, no system ssh dependency:
```go
func sshCmd() *cobra.Command {
    return &cobra.Command{
        Use:   "ssh <vm>",
        Short: "open a shell in a VM",
        Args:  cobra.ExactArgs(1),
        RunE: func(cmd *cobra.Command, args []string) error {
            cfg, creds := loadConfig()
            vmName := args[0]
            gatewayHost := cfg.GatewayAddr  // default: "spwn.run:2222"

            config := &gossh.ClientConfig{
                User: vmName,
                Auth: []gossh.AuthMethod{
                    gossh.Password(creds.Token), // bearer token as password
                },
                HostKeyCallback: knownHostsCallback(cfg),
            }
            conn, err := gossh.Dial("tcp", gatewayHost, config)
            // open session, request pty, set terminal to raw mode, relay I/O
        },
    }
}
```

## gotchas

- **platform key bootstrap**: the host agent logs the public key on first run; operator
  must add it to the rootfs's `/root/.ssh/authorized_keys` before SSH relay works
- **pty dimensions**: need to relay window resize events (SIGWINCH) from the user's
  terminal through the gateway's SSH channel → gRPC → host agent → VM SSH channel
- **known hosts**: `spwn ssh` caches the gateway's host key in `~/.config/spwn/known_hosts`
  to avoid TOFU prompts on reconnect; keyed by `spwn` alias not the raw IP
- **bearer token as password**: the control plane's `/internal/gateway/auth/password`
  must accept BOTH account password AND a valid api_token — `get_account_id_by_token_hash`
  already exists in db; just try both
- **host agent address**: when the gateway calls `GET /internal/gateway/vm`, the control
  plane must return the host agent's gRPC address (from the `hosts` table) so the gateway
  knows which agent to call StreamConsole on
- **concurrent streams**: each SSH session opens one StreamConsole gRPC stream; host
  agent must handle multiple concurrent streams per VM gracefully (multiple users SSH
  into the same VM = multiple shells, which is fine)
- **jailer network isolation**: the host agent must reach the VM at `172.16.x.2:22` from
  the host. iptables rules may need a specific ACCEPT rule for agent → VM SSH traffic
  (port 22 on the guest IP) if the current rules only allow outbound from VMs, not inbound
  from host
